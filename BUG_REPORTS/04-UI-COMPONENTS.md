# Баги: UI компоненты и уведомления

## BUG-007: Дублирование сообщений об успехе при создании навыков

**Приоритет**: Low  
**Модуль**: UI компоненты / Уведомления

### Описание
При создании навыка сообщение "Навык успешно создан." отображается дважды (дублируется). Это та же проблема, что и BUG-004, но проявляется при создании навыков.

### Шаги для воспроизведения
1. Авторизоваться в систему (роль: admin)
2. Перейти в раздел управления навыками
3. Создать новый навык
4. Наблюдать за сообщением об успехе

### Ожидаемое поведение
- Сообщение "Навык успешно создан." должно отображаться один раз

### Фактическое поведение
- Сообщение отображается дважды (дублируется)
- Видны два идентичных уведомления с зеленым фоном

### Файлы для проверки
- `resources/js/Components/Shared/FlashMessage.vue`
- `resources/js/Layouts/AdminLayout.vue` (если используется)
- Страницы создания навыков
- Любые страницы, где может быть дублирование flash-сообщений

### Предполагаемая причина
Та же проблема, что и в BUG-004:
1. Компонент `FlashMessage` включен в layout
2. На странице также есть встроенный блок для отображения flash-сообщений
3. Оба блока отображают одно и то же сообщение

### Решение
1. Убедиться, что все страницы используют единый компонент `FlashMessage` из layout
2. Удалить все встроенные блоки flash-сообщений со страниц
3. Проверить все страницы админ-панели на наличие дублирующих сообщений

## BUG-008: Проблема с отображением flash-сообщений в разных layouts

**Приоритет**: Medium  
**Модуль**: UI компоненты / Layouts

### Описание
Разные layouts могут иметь разные подходы к отображению flash-сообщений, что приводит к несогласованности и дублированию.

### Файлы для проверки
- `resources/js/Layouts/ClientLayoutWithSidebar.vue` (строка 33)
- `resources/js/Layouts/AdminLayout.vue`
- `resources/js/Layouts/StudentLayout.vue`
- `resources/js/Layouts/PartnerLayout.vue`
- `resources/js/Components/Shared/FlashMessage.vue`

### Предполагаемая причина
Не все layouts используют единый компонент `FlashMessage`. Некоторые могут иметь встроенные блоки или вообще не отображать сообщения.

### Решение
1. Убедиться, что все layouts используют единый компонент `FlashMessage`
2. Создать единую систему отображения flash-сообщений
3. Удалить все дублирующие блоки сообщений

## BUG-009: Flash-сообщения могут не очищаться правильно

**Приоритет**: Low  
**Модуль**: UI компоненты / FlashMessage

### Описание
Компонент `FlashMessage` использует автоматическое закрытие через 3.5 секунды, но могут быть проблемы с очисткой сообщений, если пользователь быстро переключается между страницами.

### Файлы для проверки
- `resources/js/Components/Shared/FlashMessage.vue`

### Предполагаемая причина
При быстром переключении между страницами flash-сообщения могут оставаться в props и не очищаться корректно.

### Решение
Проверить логику очистки сообщений в `FlashMessage.vue` и убедиться, что при переходе на новую страницу старое сообщение корректно удаляется.
